# Markdown 全格式详解与实例演示

## 一、标题格式（Headings）

Markdown 支持 6 级标题，使用 `#` 符号开头，`#` 数量对应标题级别，`#` 与标题文本之间需保留一个空格。

# 1 级标题（最大）

## 2 级标题

### 3 级标题

#### 4 级标题

##### 5 级标题

###### 6 级标题（最小）

## 二、文本基础样式

通过特殊符号可实现文本的加粗、斜体、删除线、下划线等效果，满足不同强调需求。

### 1. 斜体（Italic）

使用 `*` 或 `_` 包裹文本，两种方式效果一致。



* 示例 1：`*这是斜体文本*` → *这是斜体文本*

* 示例 2：`_这也是斜体文本_` → *这也是斜体文本*

### 2. 加粗（Bold）

使用 `**` 或 `__` 包裹文本，两种方式效果一致。



* 示例 1：`**这是加粗文本**` → **这是加粗文本**

* 示例 2：`__这也是加粗文本__` → **这也是加粗文本**

### 3. 加粗 + 斜体（Bold + Italic）

组合使用 `***` 或 `___` 包裹文本。



* 示例：`***这是加粗且斜体的文本***` → *这是加粗且斜体的文本*

### 4. 删除线（Strikethrough）

使用 `~~` 包裹文本，适用于标注已废弃或修正的内容。



* 示例：`~~这是被删除的文本~~` → ~~这是被删除的文本~~

### 5. 下划线（Underline）

Markdown 原生不支持下划线，需借助 HTML 标签 `<u>` 实现。



* 示例：`<u>这是带下划线的文本</u>` → 这是带下划线的文本

### 6. 行内代码（Inline Code）

使用 `` ` ``（反引号，位于键盘左上角 `Esc` 下方）包裹代码片段，适用于展示短代码或命令。



* 示例：`print("Hello, Markdown!")` 是 Python 中的打印语句。

## 三、列表（Lists）

Markdown 支持有序列表和无序列表，可嵌套使用，满足内容分层需求。

### 1. 无序列表（Unordered List）

使用 `-`、`*` 或 `+` 作为列表项标记，标记与文本之间需保留一个空格，三种标记效果一致。



* 水果


  * 苹果

  * 香蕉

  * 橙子

* 蔬菜


  * 番茄

  * 黄瓜

* 主食


  * 米饭

  * 面条

### 2. 有序列表（Ordered List）

使用数字 + `.` 作为列表项标记，标记与文本之间需保留一个空格，数字顺序不影响最终渲染结果（但建议按逻辑排序）。



1. 第一步：打开 Markdown 编辑器

2. 第二步：输入文本内容

3. 第三步：使用 Markdown 格式排版


   1. 子步骤 1：设置标题

   2. 子步骤 2：添加列表

4. 第四步：预览或导出文档

## 四、引用（Blockquotes）

使用 `>` 作为引用标记，可嵌套引用（多组 `>`），适用于引用他人观点、文献内容或重要提示。

### 1. 基础引用

> Markdown 是一种轻量级标记语言，由约翰・格鲁伯（John Gruber）于 2004 年创建，旨在让人们用易读易写的纯文本格式编写文档，同时可轻松转换为 HTML 等格式。

### 2. 嵌套引用

> 第一层引用

> 第二层引用（嵌套在第一层内）

> 第三层引用（嵌套在第二层内）

回到第一层引用，引用内可换行（空行 + `>`）

### 3. 引用内混排格式

引用内可结合标题、列表、代码等格式：

### 引用内的标题



* 引用内的无序列表项 1

* 引用内的无序列表项 2

  引用内的行内代码：`import pandas as pd`



## 五、代码块（Code Blocks）

适用于展示多行代码或命令，支持语法高亮（需指定语言），有两种实现方式。

### 1. 缩进式代码块（简单版）

在代码前添加 4 个空格或 1 个制表符（Tab），但不支持语法高亮。



```
\# 这是缩进式代码块（注意：实际使用时无需加 \`\`\`，只需缩进）

def add(a, b):

&#x20;   return a + b

result = add(10, 20)

print(result)
```

### 2. 围栏式代码块（推荐版）

使用三组 `` ` `` 或三组 `~` 包裹代码，在开头的三组符号后指定语言名称，可实现语法高亮，支持 Python、Java、HTML、CSS、JavaScript 等主流语言。

#### Python 代码示例



```
\# 计算斐波那契数列前 10 项

def fibonacci(n):

&#x20;   fib = \[0, 1]

&#x20;   while len(fib) < n:

&#x20;       next\_num = fib\[-1] + fib\[-2]

&#x20;       fib.append(next\_num)

&#x20;   return fib

\# 调用函数并打印结果

fib\_result = fibonacci(10)

print("斐波那契数列前 10 项：", fib\_result)
```

#### HTML 代码示例



```
\<!DOCTYPE html>

\<html lang="zh-CN">

\<head>

&#x20;   \<meta charset="UTF-8">

&#x20;   \<title>Markdown 演示页面\</title>

&#x20;   \<style>

&#x20;       body { font-family: Arial, sans-serif; }

&#x20;       h1 { color: #2c3e50; }

&#x20;   \</style>

\</head>

\<body>

&#x20;   \<h1>Hello, Markdown!\</h1>

&#x20;   \<p>这是 HTML 代码块示例。\</p>

\</body>

\</html>
```

## 六、链接（Links）

Markdown 支持行内链接、参考链接和锚点链接，满足不同场景下的跳转需求。

### 1. 行内链接（Inline Links）

格式：`[链接文本](链接地址 "可选的链接标题")`，鼠标 hover 链接时会显示标题。



* 示例：访问 [Markdown 官方文档](https://daringfireball.net/projects/markdown/) 了解更多细节。

### 2. 参考链接（Reference Links）

适用于同一链接多次使用的场景，先定义链接标识，再在文档任意位置引用，格式更清晰。



* 定义链接：`[链接标识]: 链接地址 "可选标题"`（可放在文档末尾，不影响渲染）

* 引用链接：`[链接文本][链接标识]`

#### 示例



* 推荐使用 [Typora](https://typora.io/) 或 [VS Code](https://code.visualstudio.com/) 编辑 Markdown 文档。

* [GitHub](https://github.com/) 也支持 Markdown 格式，常用于项目 README 文件。

### 3. 锚点链接（Anchor Links）

用于文档内部跳转（如跳转到指定标题），格式：`[跳转文本](#标题内容)`，注意：



* 标题内容需小写，空格替换为 `-`，特殊字符（如 `#`、`!`）需省略或转义。

* 示例：点击跳转到 [代码块](#五代码块code-blocks) 部分。

## 八、表格（Tables）

使用 `|` 分隔列，`-` 分隔表头与内容，支持对齐方式设置，格式简洁但需注意对齐。

### 1. 基础表格



* 第一行：表头（列名），用 `|` 分隔

* 第二行：分隔线，`-` 数量至少 1 个，可结合 `:` 设置对齐方式

* 第三行及以后：表格内容，用 `|` 分隔

#### 示例



| 姓名 | 年龄 | 职业   |
| -- | -- | ---- |
| 张三 | 25 | 程序员  |
| 李四 | 30 | 产品经理 |
| 王五 | 28 | 设计师  |

### 2. 带对齐方式的表格



* `:` 放在 `-` 左侧：左对齐

* `:` 放在 `-` 右侧：右对齐

* `:` 放在 `-` 两侧：居中对齐

#### 示例



| 水果 | 单价（元 / 斤） | 库存（斤） |
| -- | --------- | ----- |
| 苹果 | 5.8       | 100   |
| 香蕉 | 3.2       | 150   |
| 橙子 | 4.5       | 80    |



* 说明：“水果” 列左对齐，“单价” 列右对齐，“库存” 列居中对齐。

## 九、分割线（Horizontal Rules）

使用 3 个及以上的 `-`、`*` 或 `_` 单独成行，可用于分隔文档不同章节，三种符号效果一致。



* 示例 1（`-`）：



***



* 示例 2（`*`）：



***



* 示例 3（`_`）：



***

注意：分割线前后建议空行，避免与标题格式冲突（如 `---` 可能被识别为 2 级标题的分隔线）。

## 十、脚注（Footnotes）

用于添加文档注释或补充说明，不影响正文阅读，格式：



* 定义脚注：`[^脚注标识]`（放在需要注释的文本后）

* 编写脚注内容：`[^脚注标识]: 脚注具体内容`（可放在文档末尾）

### 示例

Markdown 支持多种扩展语法，如数学公式、流程图等 \[^1]，不同编辑器对扩展语法的支持程度可能不同 \[^2]。

\[^1]: 例如，使用 MathJax 可在 Markdown 中插入 LaTeX 数学公式，格式为 `$数学公式$`（行内公式）或 `$$数学公式$$`（块级公式）。

\[^2]: 常见的支持扩展语法的编辑器有 Typora、Obsidian、Notion 等，部分编辑器需手动开启扩展功能。

## 十一、任务列表（Task Lists）

用于标记任务完成状态，格式：`- [ ] 未完成任务`、`- [x] 已完成任务`（`x` 不区分大小写），支持嵌套。
